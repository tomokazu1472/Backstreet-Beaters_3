# Backstreet Beaters

**リズムに合わせて消しまくれ！リズムアクション3マッチパズル**  
Unity / C# による個人制作（初作品） | 制作期間：2025年4月〜2025年10月（約6ヶ月）

---

## ゲーム概要

音ゲーの楽しさと落ちものパズルを組み合わせた作品です。ヒップホップをテーマに、リズム感を大切にしたゲームデザインを目指しました。

盤面上のボールを入れ替えて同じ色を3つ以上揃えて消すマッチ3パズルで、BGMのビートに合わせてボールのスポーンや消去が進行します。

- ステージごとに異なるBGM・難易度・ミッションが設定されている
- 特定の並び方でマッチさせると、アイテムボールが生成され強力な効果が発動する
- コンボを繋げるほどスコア倍率が上がる
- ロゴ・アイテムなどの画像素材はすべて手描き

## システム設計

### ビート同期システム

ゲームの核となる仕組みです。`AudioSettings.dspTime`（サンプル精度の時刻）を使い、BGMのBPMから次のビートタイミングを算出します。`FixedUpdate`で毎フレーム監視し、ビートが来たタイミングで盤面の処理（マッチ判定・スポーン・アイテム発動）を発火させます。

BGMの一時停止・再開時には停止していた時間を`_bgmStartTime`に加算することでビートカウントのズレを補正し、シームレスなBGMループのための「ループポイントへのビートワープ」機能も実装しています。

```
BeatCtrl（ビート計測）
    └─ BeatAction.DoBeatAction()
          ├─ MatchCheck（マッチ判定）
          ├─ DeleteCtrl（ボール削除・アイテム生成）
          ├─ SpawnCtrl（新規ボールスポーン）
          └─ ItemSpawn（アイテム発動）
```

### ボール管理

フィールドを`Ball[,]`の2次元配列で管理しています。論理座標（`Vector2Int`）と表示座標（Transform）を分離し、論理座標はビートと同時に更新、表示座標はDOTweenでアニメーションさせる設計です。これにより、アニメーション中でも次の盤面状態を正確に計算できます。

### アイテムシステム

`Ball`クラスを基底クラスとして、各アイテムを継承で実装しています。

```
Ball（基底クラス）
 ├─ CanSide    → 横一列消去
 ├─ CanUpDown  → 縦一列消去
 ├─ Record     → エリア消去
 └─ MirrorBall → フィールド上の特定色をすべて消去
```

`SetActive()` / `Activate()` を仮想メソッドで定義し、各アイテムがオーバーライドする形をとっています。アイテムが「発動待ち状態」になるとビートのタイミングで順次 `Activate()` が呼ばれます。

### マッチ判定

横・縦を独立してスキャンし、同じ`ItemId`が3つ以上連続するボールを`HashSet<Ball>`に収集します。次に`MatchShape`でマッチした形状（3個一直線・L字・T字など）を判定し、形状に応じたアイテムIDを決定します。

### ステージデータ

ステージ情報は`StageInfo`（ScriptableObject）で定義します。BGMデータ（BPM・ループポイント・ミュートビートリストなど）も`GameBGM`クラスとしてシリアライズし、エディタ上で各ステージを独立して設定できます。

## こだわった点

**ステージ選択のUIアニメーション**  
ステージ選択画面の切り替えアニメーション、特にボタン選択時のアニメーションに約1週間を費やしました。音楽・リズムをテーマにしたゲームである以上、UIの動きにもリズム感が出るよう細部まで調整しています。

**ビート同期の精度**  
`Update()`ではフレームレートによって呼び出しタイミングがズレるため、`FixedUpdate()`でビートを検知し、`WaitForSeconds`で残り時間だけ待ってから処理を実行する方式を採用しました。

**論理座標と表示座標の分離**  
アニメーション中に次の処理が始まっても盤面の状態が正しく保たれるよう、移動アニメーションはDOTweenに任せ、配列上の座標は即時更新する設計にしました。

**アイテムの拡張性**  
新しいアイテムを追加する際に既存コードをほぼ変更しなくてよいよう、`Ball`の継承と仮想メソッドによる設計を採用しました。
